Authentification

En-têteDescription
X-Intranet-TokenToken partagé (INTRANET_SHARED_TOKEN)
X-UI-SessionSession signée (réponse ui-login / ui-session)
X-Actor-UsernameAudit (optionnel)
X-Actor-Typeinternal, admin, public
X-Public-EmailCompte public (destinations propres)

Sans auth : /health, /ready, /version, UI statique, ui-config, ui-login, ui-session, GET /releases/recent?public=true.

Exemples complets d'appels API

Base : http://192.168.100.30:3088 — remplacez VOTRE_TOKEN par INTRANET_SHARED_TOKEN. Les réponses sont abrégées (…) lorsque le schéma est déjà documenté plus haut.

Matrice des acteurs
aucun catalogue public, santé
public + X-Public-Email — ses abonnements/destinations uniquement
internal + token — intranet, scripts, opérateur
admin + token ou session UI — console /admin

6. Cas particuliers et erreurs

401 — Token manquant ou invalide

curl -s http://127.0.0.1:3088/api/v1/iso-items

Réponse 401

{ "error": "unauthorized" }

403 — Acteur insuffisant

internal appelle une route admin

curl -s http://127.0.0.1:3088/api/v1/admin/overview \
  -H "X-Intranet-Token: VOTRE_TOKEN" \
  -H "X-Actor-Type: internal"

Réponse 403

{ "error": "admin_required" }

public sans X-Public-Email sur une destination d'un autre user

{ "error": "forbidden" }

403 — Actions publiques désactivées

curl -s -X POST http://127.0.0.1:3088/api/v1/scans/run \
  -H "X-Intranet-Token: VOTRE_TOKEN" \
  -H "X-Actor-Type: internal"

Si PUBLIC_UI_ALLOW_ACTIONS=false et acteur non admin

{ "error": "public_actions_disabled" }

409 — Vérification liens déjà en cours

{ "error": "link_check_already_running" }

502 — Échec téléchargement local

curl -s -X POST http://127.0.0.1:3088/api/v1/releases/99/download \
  -H "X-Intranet-Token: VOTRE_TOKEN" \
  -H "X-Actor-Type: admin"

Réponse 502

{
  "ok": false,
  "error": "download_failed",
  "message": "HTTP 404"
}

Le message réel est dans message, pas dans error.

502 — Échec envoi notification (test destination)

curl -s -X POST http://127.0.0.1:3088/api/v1/destinations/8/test \
  -H "X-Intranet-Token: VOTRE_TOKEN" \
  -H "X-Actor-Type: public" \
  -H "X-Public-Email: user@example.com" \
  -H "Content-Type: application/json" \
  -d '{ "message": "Test" }'

Réponse 502

{
  "error": "delivery_failed",
  "channel": "discord_webhook",
  "code": "ENOTFOUND",
  "message": "getaddrinfo ENOTFOUND discord.com"
}

400 — Validation métier

{ "error": "discovery_regex_required" }
{ "error": "email_required" }
{ "error": "iso_item_ids_required" }

404 — Ressource absente

{ "error": "iso_item_not_found" }
{ "error": "release_not_found" }
{ "error": "user_not_found" }
{ "error": "local_file_not_found" }

409 — Suppression utilisateur public avec abonnements actifs

curl -s -X DELETE "http://127.0.0.1:3088/api/v1/public/users/by-email?email=user@example.com" \
  -H "X-Intranet-Token: VOTRE_TOKEN"

Réponse 409

{ "error": "user_has_active_subscriptions" }
